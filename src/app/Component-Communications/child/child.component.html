Child to Parent Communication (Using @Output and EventEmitter)

Child-to-parent communication in Angular involves passing data or triggering actions from a child component to its parent. This is a common pattern in component-based architectures like Angular, where components are often hierarchically structured. There are several techniques to achieve child-to-parent communication, including @Output() decorators, services, and view queries.

Use Case: When a child component needs to send data or events back to the parent.

The child component defines an @Output() property, which is an EventEmitter.

The parent component listens to this event and handles the emitted data.
