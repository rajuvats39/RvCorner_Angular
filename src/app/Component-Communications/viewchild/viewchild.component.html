Using @ViewChild for Parent-Child Communication

Use Case: When the parent component needs direct access to the child component's methods or properties.

The parent component uses the @ViewChild() decorator to get a reference to the child component instance.

@ViewChild is a decorator that provides access to child components, directives, or DOM elements from the parent component. It allows you to interact with child components and their properties, methods, or DOM elements, enabling parent-child component communication.

How @ViewChild Works

Purpose: To access and manipulate a child component, directive, or HTML element from the parent component.

Usage: It is used when the parent needs to call methods or access properties of a child component.
