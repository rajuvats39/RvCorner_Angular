Dependency Injection
Dependency Injection (DI) is a technique in which a class receives its dependencies from external sources rather than creating them itself.

Dependency Injection (DI) is a design pattern in which a class receives its dependencies from external sources rather than creating them itself. Angularâ€™s DI system allows you to inject services, other components, or third-party libraries into your Angular classes (components, directives, services, etc.).

Why Use Dependency Injection?:

Inversion of Control: Instead of the class being responsible for creating its dependencies, the dependencies are provided by an external system (the Angular injector).

Modular and maintainable code: Components and services become more modular because they rely on external inputs.

Testability: Dependencies can be mocked or replaced with test-specific implementations, making unit testing easier.

Singletons: DI ensures that a single instance of a service can be shared across the application or between components, avoiding unnecessary creation of objects.

Injector

Injector holds the Providers and is responsible for resolving the dependencies and injecting the instance of the Dependency to the Consumer.

The Injector uses Injection Token to search for Dependency in the Providers. It then creates an instance of the dependency and injects it into the consumer

It is responsible forcreating and managing the instances of services and injecting them where required.

Key Feature

Creating Services: The Injector is responsible for creating instances of services when they are requested, and it also manages their lifecycle, ensuring that the same instance is reused if it's marked as a singleton.

Providing Dependencies: The Injector supplies the necessary dependencies (services, tokens) to components, directives, or other services when required. For instance, if a component needs a service, the injector provides that service.
