Services

A service in Angular is a class that contains reusable business logic or functionality that can be shared across different components. It is primarily used to encapsulate common functionalities like fetching data from an API, logging, authentication, utility functions, and much more.

Our components need to access the data. You can write data access code in each component, but that is very inefficient and breaks the rule of single responsibility. The Component must focus on presenting data to the user. The task of getting data from the back-end server must be delegated to some other class. We call such a class a Service class. Because it provides the service of providing data to every component that needs it.

01) Services allow us to create reusable code and use it for every component that needs it.

02) The Services can be injected into components and other services using the dependency injection system.

03) The dependencies are declared in the Module using the Providerâ€™s metadata.

04) The Angular creates a tree of injectors & Providers that resembles the Component Tree. This is called the hierarchical pattern.

05) Service is a piece of reusable code with a focused purpose. A code that you will use in many components across your application

Services are typically designed to:

Promote code reusability: Common logic can be abstracted into services so it can be reused in multiple places.

Encapsulate logic: Business logic is separated from components to keep components clean and focused on view logic.

Facilitate testing: Services can be unit tested independently of components.

Enable Dependency Injection: Services are injected into components, other services, or directives to provide their functionality.

Features that are independent of components such a logging services

Share logic or data across components

Encapsulate external interactions like data access
