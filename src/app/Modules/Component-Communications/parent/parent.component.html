Parent to Child Communication (Using @Input)

Parent-to-child communication is a common pattern where a parent component passes data to its child components. Angular provides the@Input() decorator to facilitate this communication. This allows the child component to receive data dynamically, enabling components to work together efficiently.

Use Case

01)When a parent component needs to pass data to its child component.

02)The child component defines an @Input() property to receive the data from the parent.

03)The parent component binds the data to the child component's property using property binding in the template.

04)The @Input() decorator in the child component marks a property as a target for receiving data from the parent.

05)The parent component passes data to the child via property binding ([childProperty]="parentData").

Key Points to Remember:

@Input Data Binding:

The parent can bind data to the child component using the square brackets ([]) syntax in the template. Change Detection

Angular automatically tracks changes to the @Input() properties and updates the child component when the parent changes the value.

Type Safety:

You can type your @Input() properties for better type safety, ensuring only valid data types are passed between components.

Limitations:

@Input() should be used only for one-way communication from parent to child. For two-way data binding, use@Output() to send data back from child to parent. Conclusion:

The@Input() decorator is a powerful tool in Angular for passing data from parent to child components, whether it's simple data types or complex objects and arrays. Understanding how @Input() works and integrating it with lifecycle hooks like ngOnChanges allows for dynamic, flexible component communication
