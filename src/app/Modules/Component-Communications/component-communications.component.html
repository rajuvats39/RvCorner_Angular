Component Communications

Component communication in Angular involves passing data between different components. This is essential for building dynamic applications where components need to share information or notify one another of state changes. There are several ways to achieve component communication in Angular, depending on the relationship between the components

Way Component Communications

1. Parent to Child Communication (Using @Input)

Use Case: When a parent component needs to pass data to its child component.

The child component defines an @Input() property to receive the data from the parent.

The parent component binds the data to the child component's property using property binding in the template.

1.2. Parent to Child Local References in Templates

Use Case: A simple way to pass data or interact with a child component in the template without using Angular decorators.

You define a local template reference in the parent component template, then access the child component's methods or properties.

1.3. Parent to Child Using @ContentChild and @ContentChildren

Use Case: A simple way to pass data or interact with a child component in the template without using Angular decorators.

You define a local template reference in the parent component template, then access the child component's methods or properties.

1.4. Using @ViewChild for Parent-Child Communication

Use Case: When the parent component needs direct access to the child component's methods or properties.

The parent component uses the @ViewChild() decorator to get a reference to the child component instance.

2. Child to Parent Communication (Using @Output and EventEmitter)

Use Case: When a child component needs to send data or events back to the parent.

The child component defines an @Output() property, which is an EventEmitter.

The parent component listens to this event and handles the emitted data.

3. Sibling to Sibling Communication or Unrelated Components(Service with RxJS/Subjects)

Use Case: When two sibling components need to communicate.

A shared service can be used to facilitate communication between sibling components. This service can use RxJS Subject or BehaviorSubject to pass data between components.
