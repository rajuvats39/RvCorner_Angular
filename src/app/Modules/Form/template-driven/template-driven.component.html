Template-driven form

In Template Driven Forms we specify behaviors/validations using directives and attributes in our template and let it work behind the scenes. All things happen in Templates hence very little code is required in the component class. This is different from the reactive forms, where we define the logic and controls in the component class.

Points to Setup Template-driven form

1. Import FormsModule in AppModule.

2. Forms are set up using the ngForm directive.
2.1. The ngForm is a directive which automatically convert Angular form in to Template-driven form.
2.2. Binds itself to the Form directive.
2.3. Creates a top-level FormGroup instance.
2.4. CreatesFormControl instance for each of child control, which has ngModel directive..
2.5. CreatesFormGroup instance for each of the NgModelGroup directive.

3. Controls are set up using the ngModel directive.
3.1. ngModel will use the name attribute to create the FormControl instance for each of the Form field it is attached.
3.2. ngModel also provides the two-way data binding.

4. Forms are submit using the ngSubmit event.
5. The Validations are configured in the template via directives.

Advantages

1. Contains little code in the component class.
2. Easier to set up.

DisAdvantages

1. Difficult to add controls dynamically.
2. Unit testing is a challenge.
